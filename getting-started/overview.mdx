This document describes the zkPass oracle protocol from a technical perspective, detailing the cryptographic components and core building blocks that constitute it.

The zkPass protocol is designed as a private data oracle, leveraging **Vector Oblivious Linear Evaluation (VOLE)** to construct efficient commitments, known as **VOLE-based Zero-Knowledge Proofs (VOLE-ZK)**. Beyond its use of SoftSpokenOT for Oblivious Transfer (OT), protocol security relies critically on the robustness of the **VOLE-in-the-Head (VOLEitH)** technique.

zkPass operates in **Hybrid Mode**, combining **Proxy Mode** and **MPC Mode**. This allows the protocol to adapt to different network conditions and server restrictions without compromising security or performance.
