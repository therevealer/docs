## Purpose & Vision

The **Verifiable Reputation Score (VRS)** is a proof-based mechanism designed to recognise authenticity, contribution, and engagement across the zkPass ecosystem.

It quantifies the completeness and credibility of a user’s digital identity across multiple verified data sources, while preserving privacy through zero-knowledge proofs.

Higher scores grant greater access and priority in ecosystem participation, ensuring that credibility and genuine contribution are rewarded alongside timing.

## Core Principle

VRS is not about popularity, wealth, or social ranking.

It measures **proof density** — how many independent aspects of your digital footprint are verified and how consistent they are.

- Binance KYC → proves real-person uniqueness
- LinkedIn / GitHub → professional authenticity
- Duolingo / Kaito Yap → learning and contribution

Together, these form your **verifiable identity fabric**.

## Conceptual Design

VRS is built on zkPass’s open-source **Coverage-Penalized Multi-Source Reputation Framework**, designed to evaluate both **verified proofs** and **missing coverage**.

### Scoring Layers

| Layer                  | Purpose                     | Description                                                                       |
| ---------------------- | --------------------------- | --------------------------------------------------------------------------------- |
| Feature Normalization  | Standardize raw metrics     | Maps heterogeneous data into [0–1] using log scale, SoftClip, and boolean mapping |
| Per-Source Aggregation | Combine features per source | Weighted sum based on feature importance                                          |
| Cross-Source Weighting | Aggregate sources           | Applies global weights (e.g. KYC \> social)                                       |
| Weighted Coverage      | Measure completeness        | Computes ratio of verified vs expected sources                                    |
| Nonlinear Penalty      | Penalize gaps               | Applies exponent α \> 1 to missing coverage                                       |

## Properties

- **Bounded:** Always within [0,100]
- **Monotonic:** New proofs never reduce score
- **Weighted Fairness:** High-trust sources matter more
- **Coverage Sensitivity:** Major gaps penalized harder
- **Nonlinear Incentives:** Tunable via α
- **Auditable:** Every layer is inspectable
- **Extensible:** Supports decay, confidence, and federation

## Reference Implementation

```python
def compute_vrs(sources, alpha=1.5):
    total_weight = sum(s["weight"] for s in sources)
    provided = [s for s in sources if s["provided"]]
    coverage = sum(s["weight"] for s in provided) / total_weight
    quality = sum(s["weight"] * s["score"] for s in provided)
    return round(100 * quality * (coverage ** alpha), 2)
```
